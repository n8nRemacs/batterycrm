## 3. DOMAIN CONTEXT TYPES

Контексты, общие для ВСЕХ вертикалей домена.

### Context Type 1: device

| Поле | Значение |
|------|----------|
| **Код** | `device` |
| **Название** | `Устройство` |
| **Описание** | `Информация о телефоне/устройстве клиента` |
| **Tier** | [x] small  [ ] nano  [ ] optima |

**Что извлекаем (простым языком):**
```
Бренд телефона (Apple, Samsung, Xiaomi и т.д.)
Модель (iPhone 14 Pro, Galaxy S23)
Цвет (если упоминает)
Объём памяти (64/128/256/512 ГБ)
IMEI (если есть, 15 цифр)
```

**Возможные значения brand:**
| Значение | Описание |
|----------|----------|
| `Apple` | iPhone, iPad |
| `Samsung` | Galaxy, Note |
| `Xiaomi` | Mi, Redmi, Poco |
| `Huawei` | P, Mate, Nova |
| `Honor` | Honor |
| `Realme` | Realme |
| `OPPO` | OPPO |
| `Vivo` | Vivo |
| `Google` | Pixel |
| `OnePlus` | OnePlus |
| `Nothing` | Nothing Phone |

**Структура результата (JSON):**
```json
{
  "brand": "string | null",
  "model": "string | null",
  "color": "string | null",
  "storage": "number | null",
  "imei": "string | null"
}
```

---

### Context Type 2: device_condition

| Поле | Значение |
|------|----------|
| **Код** | `device_condition` |
| **Название** | `Состояние устройства` |
| **Описание** | `Новый, б/у, восстановленный, разбитый` |
| **Tier** | [x] nano |

**Что извлекаем:**
```
Состояние устройства: новое, б/у, разбитое
Детали состояния: царапины, трещины, идеальное
```

**Возможные значения:**
| Код | Описание | Примеры от клиента |
|-----|----------|-------------------|
| `new` | Новый, в пленке | "новый", "запечатанный" |
| `like_new` | Как новый | "идеальное состояние", "без царапин" |
| `good` | Хорошее | "хорошее состояние", "небольшие царапины" |
| `fair` | Удовлетворительное | "потёртости", "царапины на экране" |
| `poor` | Плохое, но работает | "трещины", "разбит но работает" |
| `broken` | Не работает | "не включается", "разбит полностью" |

**Структура результата (JSON):**
```json
{
  "condition": "string",
  "details": "string | null"
}
```

---

### Context Type 3: price_expectation

| Поле | Значение |
|------|----------|
| **Код** | `price_expectation` |
| **Название** | `Ожидание по цене` |
| **Описание** | `Бюджет клиента или ценовая категория` |
| **Tier** | [x] nano |

**Что извлекаем:**
```
Конкретная сумма (до 30000, около 50к)
Диапазон (от 20 до 30 тысяч)
Категория (недорого, бюджетный, флагман)
```

**Возможные значения category:**
| Код | Описание | Примеры |
|-----|----------|---------|
| `budget` | Бюджетный | "недорого", "подешевле" |
| `mid` | Средний | "средний ценник", "не дорого и не дёшево" |
| `premium` | Премиум | "премиум", "качественный" |
| `flagship` | Флагман | "флагман", "топовый" |

**Структура результата (JSON):**
```json
{
  "min": "number | null",
  "max": "number | null",
  "category": "string | null"
}
```

## 5. ПРИМЕРЫ СООБЩЕНИЙ

**Пример 1:**
```
Привет! У меня айфон 14 про, разбился экран
```
Ожидаемый контекст:
```json
{
  "device": {"brand": "Apple", "model": "iPhone 14 Pro"},
  "device_condition": {"condition": "broken", "details": "разбит экран"}
}
```

**Пример 2:**
```
Ищу самсунг до 30 тысяч, б/у можно
```
Ожидаемый контекст:
```json
{
  "device": {"brand": "Samsung"},
  "price_expectation": {"max": 30000, "category": "budget"},
  "device_condition": {"condition": "good"}
}
```

**Пример 3:**
```
Нужен дисплей на редми ноут 12
```
Ожидаемый контекст:
```json
{
  "device": {"brand": "Xiaomi", "model": "Redmi Note 12"}
}
```
